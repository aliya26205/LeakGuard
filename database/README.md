# 🗄️ LeakGuard Database

LeakGuard uses **Supabase (PostgreSQL)** as its backend database to securely store employee information, security policies, activity logs, and alerts generated by the Chrome Extension.

---

# Database Technology

- **Database:** PostgreSQL
- **Platform:** Supabase
- **Schema:** Public

---

# Database Tables

The database contains the following tables:

| Table Name | Description |
|------------|-------------|
| **admins** | Stores administrator login information. |
| **employees** | Stores employee details and activation information. |
| **policies** | Stores configurable security policies used for leak detection. |
| **activity_logs** | Stores every detected leak attempt and user activity. |
| **alerts** | Stores alerts generated from activity logs for administrator review. |

# Database Setup

## Step 1

Create a new project in **Supabase**.

https://supabase.com

---

## Step 2

After creating the project, open:

```
SQL Editor
```

---

## Step 3

Open the file:

```
database/schema.sql
```

Copy the entire SQL script.

---

## Step 4

Paste the SQL into the Supabase SQL Editor.

Click:

```
Run
```

All required tables will be created automatically.

---

# Environment Configuration

After the database is created, create a file named:

```
backend/.env
```

Copy the contents from:

```
backend/.env.example
```

Then replace the placeholder values with your own:

```env
PORT=5000

SUPABASE_URL=YOUR_SUPABASE_URL

SUPABASE_SERVICE_ROLE_KEY=YOUR_SUPABASE_SERVICE_ROLE_KEY

EMAIL_USER=YOUR_EMAIL@gmail.com

EMAIL_PASS=YOUR_GMAIL_APP_PASSWORD
```

# Notes

- LeakGuard uses UUIDs as primary keys.
- Foreign key constraints maintain data integrity.
- PostgreSQL arrays are used to store multiple detected sensitive items.
- All timestamps are automatically generated using `CURRENT_TIMESTAMP`.
- Security policies can be managed dynamically from the Admin Dashboard without modifying the extension code.

---

# Security

For security reasons, the following files are **NOT** included in this repository:

- `.env`
- Supabase Service Role Key
- Gmail App Password
- Any production credentials

Use your own Supabase project and create a local `.env` file before running the application.
